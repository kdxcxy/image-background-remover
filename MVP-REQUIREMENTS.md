## 一、项目概述

### 1.1 产品定位
一个简洁高效的在线图片背景去除工具，用户无需注册，上传图片即可一键去除背景并下载结果。

### 1.2 核心价值
- **零门槛**：无需注册登录，打开即用
- **速度快**：全球 CDN 分发，秒级响应
- **免费额度**：利用 Remove.bg 免费额度，降低初期成本
- **隐私友好**：图片不落地存储，内存处理即销毁

### 1.3 MVP 目标
验证产品可行性，收集用户反馈，为后续迭代提供数据支撑。

## 三、核心功能（MVP 范围）

### 3.1 功能清单

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 图片上传 | P0 | 支持拖拽上传、点击上传，限制格式和大小 |
| 背景去除 | P0 | 调用 Remove.bg API，一键去背景 |
| 结果预览 | P0 | 并排展示原图与处理结果 |
| 图片下载 | P0 | 下载去背景后的 PNG 图片 |
| 错误提示 | P0 | 上传失败、处理失败等友好提示 |
| 移动端适配 | P1 | 响应式设计，支持手机浏览器使用 |

### 3.2 功能详情

#### 3.2.1 图片上传

**支持格式：** PNG、JPG、JPEG、WEBP
**文件大小限制：** 最大 10MB（Remove.bg 支持 25MB，留余量）
**上传方式：**
- 拖拽上传
- 点击选择文件
- 粘贴图片（Ctrl+V）

**前端校验：**
- 文件格式校验
- 文件大小校验
- 图片尺寸提示（建议 < 25MP）

#### 3.2.2 背景去除

**处理流程：**
1. 前端将图片转为 Base64
2. 发送至 Cloudflare Workers API
3. Workers 调用 Remove.bg API
4. 返回处理后的图片数据
5. 前端展示结果

**超时处理：** 30 秒超时，提示用户重试

#### 3.2.3 结果预览

**展示方式：**
- 左右并排对比（原图 vs 去背景后）
- 透明背景显示棋盘格
- 支持缩放查看细节

#### 3.2.4 图片下载

**下载格式：** PNG（透明背景）
**文件命名：** `removed_bg_原文件名.png`

## 四、技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户浏览器                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Cloudflare Pages (静态前端)                     │   │
│  │  - React + TypeScript                           │   │
│  │  - Tailwind CSS                                 │   │
│  └─────────────────────────────────────────────────┘   │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP API
                     ▼
┌─────────────────────────────────────────────────────────┐
│              Cloudflare Workers (边缘计算)              │
│  ┌─────────────────────────────────────────────────┐   │
│  │  API 端点:                                       │   │
│  │  POST /api/remove-background                    │   │
│  │  - 接收 Base64 图片                              │   │
│  │  - 调用 Remove.bg API                           │   │
│  │  - 返回处理结果                                  │   │
│  └─────────────────────────────────────────────────┘   │
└────────────────────┬────────────────────────────────────┘
                     │ HTTPS
                     ▼
┌─────────────────────────────────────────────────────────┐
│                   Remove.bg API                         │
│  - 专业背景去除服务                                      │
│  - 50 次/月免费额度                                     │
│  - 付费 $0.20/张 起                                     │
└─────────────────────────────────────────────────────────┘
```

### 4.2 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端框架 | React 18 + TypeScript | 生态成熟，开发效率高 |
| 样式方案 | Tailwind CSS | 快速开发，响应式友好 |
| 构建工具 | Vite | 开发体验好，打包快 |
| 后端运行时 | Cloudflare Workers | 全球边缘部署，无需服务器 |
| 图片处理 | Remove.bg API | 效果好，接入简单 |
| 部署平台 | Cloudflare Pages + Workers | 一体化部署，免费额度充足 |

### 4.3 数据流

```
1. 用户上传图片
   ↓
2. 前端校验 + 压缩（如需）
   ↓
3. 转为 Base64，POST 到 /api/remove-background
   ↓
4. Workers 接收请求，转发至 Remove.bg
   ↓
5. Remove.bg 返回处理后的图片
   ↓
6. Workers 返回图片数据给前端
   ↓
7. 前端展示结果，用户下载
```

## 六、非功能需求

### 6.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 页面加载 | < 2s | 首屏加载时间 |
| 图片上传 | < 3s | 10MB 图片上传 |
| 背景去除 | < 10s | Remove.bg API 响应 |
| 整体体验 | < 15s | 从上传到看到结果 |

### 6.2 可用性

- **SLA 目标：** 99% 可用性
- **监控：** Cloudflare Analytics
- **错误处理：** 友好提示，引导重试

### 6.3 安全性

| 安全措施 | 说明 |
|---------|------|
| API Key 保护 | 存储在 Workers 环境变量，不暴露前端 |
| 请求限制 | 单 IP 每分钟最多 10 次请求（防止滥用） |
| HTTPS | 全站 HTTPS 加密 |
| 无数据留存 | 图片内存处理，不落盘存储 |

### 6.4 兼容性

| 平台 | 支持范围 |
|------|---------|
| 桌面浏览器 | Chrome、Firefox、Safari、Edge 最新两个版本 |
| 移动浏览器 | iOS Safari、Android Chrome |
| 屏幕尺寸 | 响应式设计，支持 320px - 2560px |

## 八、成本估算

### 8.1 开发成本

| 项目 | 成本 |
|------|------|
| 前端开发 | 自行开发 |
| 后端开发 | 自行开发 |
| UI 设计 | 使用开源组件库（免费） |
| **开发总成本** | **时间投入约 20 小时** |

### 8.2 运营成本（月度）

| 项目 | 免费额度 | 超额费用 |
|------|---------|---------|
| Cloudflare Pages | 无限 | $0 |
| Cloudflare Workers | 100,000 请求/天 | $0.50/百万请求 |
| Remove.bg API | 50 次/月 | $0.20/张 起 |

**预估月度成本：**
- 低流量（< 50 次/月）：**$0**
- 中流量（500 次/月）：**$90**（Remove.bg）
- 高流量（5000 次/月）：**$990**（Remove.bg）

### 8.3 成本优化建议

1. **申请 Remove.bg API 提升额度**：部分用户可申请更高免费额度
2. **后续可切换方案**：流量大时考虑自建 rembg 服务
3. **用户限制**：单用户每日处理次数限制，防止滥用

## 九、风险与限制

### 9.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Remove.bg API 限流 | 用户无法处理 | 提示用户稍后重试，准备备用方案 |
| Workers 冷启动 | 首次请求慢 | 预热策略，使用 Workers KV |
| 大文件上传超时 | 处理失败 | 前端压缩，限制文件大小 |

### 9.2 业务限制

| 限制 | 说明 |
|------|------|
| 免费额度有限 | Remove.bg 仅 50 次/月，需控制使用 |
| 无用户系统 | 无法追踪用户使用情况 |
| 无历史记录 | 用户无法查看历史处理结果 |

### 9.3 合规风险

| 风险 | 应对 |
|------|------|
| 用户隐私 | 明确告知图片不存储，处理即销毁 |
| 图片版权 | 用户协议中声明用户需对上传图片负责 |
